= Site service
:toc: right
:imagesdir: images

The site service is responsible for processing requests on the endpoint `:8080/site/<project>/<branch>/*`. `<project>` and `<branch>` refer to CMS-specific projects and branches, for instance `:8080/site/default/draft`.

== Request pipeline

The site pipeline is executed as a subset of the <<../pipeline#common_request_pipeline,common pipeline>>:

image::site-pipeline.svg[Site rendering pipeline, 768px]

This section describes the standard pipeline involved in processing a page request, e.g. `:8080/site/<project>/<branch>/<path-to-page>`.

Mappings:: Uses path or content-based mappings from the site descriptor to trigger filters and HTTP functions.

Page resolver:: Looks up the content at the path and resolves its implementation and config (optionally via a page template).
+
For two content types the page resolver does not resolve any implementation:
+
* For `base:shortcut`, the referenced content path is returned with status `307 Temporary Redirect`.
* For `portal:fragment`, no implementation is needed — a fragment is rendered directly.
+
For other content types, implementations are considered in the following order:
+
. Implementation set directly on the content.
. Implementation set on a page template for the content. If the page template is unavailable (e.g. deleted), the page resolver treats it as if no page template was set.
. Implementation set on the first page template found in `\{site}/_templates` that supports the content's type.
+
If no implementation is found, the response status is `404 Not Found`.
+
NOTE: In `inline` rendering mode the status is `418` instead of `404`, so Content Studio can switch to alternative rendering. In `edit` rendering mode the status is `200` instead of `404`, to allow Content Studio to render the in-place component selector.

Page renderer:: Resolves the implementation and optional configuration (which may also come from a page template).
+
* If no implementation is found, returns `404 Not Found`.
* Otherwise invokes the implementation and returns `200 OK` by default. The output may include component placeholders.

Response processors:: For `text/html` responses, any response processors defined in the site descriptor are executed. Processors may manipulate the result, for instance by adding page contributions.

Contributions filter:: For `text/html` responses, any `pageContributions` in the response object are merged into the response body.

Response filters:: Any filters registered through a mapping execute their final processing before the response is returned from the pipeline.
