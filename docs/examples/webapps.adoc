== Minimal webapp

A webapp implementation exporting one function per HTTP method:

.src/main/resources/webapp/webapp.ts
[source,typescript]
----
const mustache = require('/lib/xp/mustache');

// Handles a GET request
exports.GET = (req) => {
    const view = resolve('my-page.html');
    const params = {
        appId: app.name,
        title: 'Hello world'
    };

    return {
        body: mustache.render(view, params),
        contentType: 'text/html'
    };
};

// Handles a POST request
exports.POST = (req) => {
    const name = req.params.name;

    return {
        body: { Hello: name },
        contentType: 'application/json'
    };
};

// Handles all other methods
exports.all = (req) => {
    if (req.method === 'DELETE') {
        handleDelete(req);
    } else if (req.method === 'PUT') {
        handlePut(req);
    }

    return {
        body: { Method: req.method },
        contentType: 'application/json'
    };
};
----

== Static website

A webapp can serve a static site — `index.html`, CSS, and other assets — by delegating to https://developer.enonic.com/docs/lib-static[lib-static]. Place the files under `src/main/resources/static/` (the library's default root):

[source]
----
src/main/resources/
├── static/
│   ├── index.html
│   └── _static/
│       └── styles.css
└── webapp/
    └── webapp.ts
----

In `index.html`, reference the stylesheet relatively:

.src/main/resources/static/index.html
[source,html]
----
<!doctype html>
<html>
  <head>
    <link rel="stylesheet" href="_static/styles.css">
    <title>Hello</title>
  </head>
  <body>
    <h1>Hello world</h1>
  </body>
</html>
----

The implementation hands every request off to `lib-static`:

.src/main/resources/webapp/webapp.ts
[source,typescript]
----
const staticLib = require('/lib/enonic/static');

exports.all = (req) => {
    return staticLib.requestHandler(req);
};
----

Once deployed, the site is reachable at `+<server>:<port>/webapp/<app-name>/+` — `index.html` is served at the root and `+_static/styles.css+` is served alongside it.
