= Admin
:toc: right

The admin service handles requests on `+/admin/*+` and powers XP's administrative user interfaces. 

== Introduction

Apps contribute two kinds of admin artifacts: *Tools* that mount in the XP admin menu, and *extensions* that plug into named interfaces exposed by the tools.

== Tools and extensions

[cols="1,3", options="header"]
|===
| Artifact | Role
| <<admin/tools#, Admin tools>> | First-class admin UIs at `+/admin/<app-key>/<tool-name>+`. Listed in the Launcher menu, gated by `role:system.admin.login` plus per-tool `allow:` rules. Content Studio is itself an admin tool.
| <<admin/extensions#, Admin extensions>> | App-provided HTTP endpoints that mount into existing admin tools at named interface points (formerly called *widgets*). Typical use: rendering an embedded UI fragment, but the response can be any HTTP payload.
|===

Tools and extensions reference each other: a tool publishes the *interface points* it exposes; an extension declares which interfaces it targets. Any extension whose interface list intersects with a tool's eligible set becomes mountable.
